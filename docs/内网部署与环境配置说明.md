# 内网部署与环境配置说明

## 1. 目标

本文档用于说明这套回归脚本在内网环境的安装与运行方式，重点覆盖：

- Python 运行时准备
- 依赖离线安装
- MySQL 连接与权限
- 运行前配置项
- 初始化、执行与验证命令


## 2. 当前实现边界

- 对比引擎：`DeepDiff(ignore_order=True)`
- 生产初始化入口（推荐）：`scripts/init_schema_only.py`
- 演示初始化入口（DEMO）：`scripts/init_demo_data.py`
- 稳定回归入口（推荐）：`scripts/regression_cli.py`（3个必填参数）
- 扩展回归入口（兼容，计划于 2026-06-30 弃用）：`scripts/run_regression_only.py`
- 目标库表初始化 SQL：`sql/init_rrs_test_dev.sql`
- 演示入口（可选）：`scripts/run_demo.py`（一键清表+造数+比对）

注意（仅 `init_demo_data.py` 涉及清表）：

- 会 `TRUNCATE` 以下表：
  - `t_compare_result_detail`
  - `t_compare_result`
  - `t_compare_rule`
  - `t_regression_batch`
  - `t_request_compare_index`
  - `t_request_info`
- 请勿在生产业务库直接执行清表操作。


## 3. 环境要求

### 3.1 系统与软件

- 操作系统：Linux 或 macOS（内网机器）
- Python：`3.10+`（已在 `3.12` 验证）
- MySQL：`8.x`（`5.7` 也可尝试，但未完整回归）

### 3.2 Python 依赖

`requirements.txt` 当前依赖：

- `deepdiff==8.6.1`
- `PyMySQL>=1.1.1`
- `mcp`（可选，仅在运行 `scripts/regression_mcp_server.py` 时需要）


## 4. 内网离线安装依赖

如果内网机器不能访问公网，建议在外网机器先下载 wheel 包。

### 4.1 外网机器准备离线包

```bash
cd /path/to/json-diff
python3 -m pip download -r requirements.txt -d wheelhouse
```

可选：打包传输

```bash
tar -czf wheelhouse.tar.gz wheelhouse
```

### 4.2 内网机器安装

```bash
cd /path/to/json-diff
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install --upgrade pip
python3 -m pip install --no-index --find-links=wheelhouse -r requirements.txt
```

如果你传的是压缩包，先解压：

```bash
tar -xzf wheelhouse.tar.gz
```


## 5. MySQL 配置要求

### 5.1 连接信息

支持两种配置方式：

1) 单串方式（兼容）：

- 变量：`MYSQL_DEMO_CONF`
- 格式：`host,port,database,user,password[,charset]`
- 示例：`127.0.0.1,3308,demo_db,dev,123456,utf8mb4`

2) 拆分方式（推荐）：

- `MYSQL_HOST`
- `MYSQL_PORT`
- `MYSQL_DATABASE`
- `MYSQL_USER`
- `MYSQL_PASSWORD`
- `MYSQL_CHARSET`

### 5.2 数据库权限

运行用户至少需要以下权限：

- 在 `demo_db` 上：`CREATE`（用于建 `rrs_test_dev` 库）
- 在 `rrs_test_dev` 上：`SELECT, INSERT, UPDATE, DELETE, CREATE, DROP, INDEX, ALTER`

如果库由 DBA 预先创建，也可以去掉 `CREATE DATABASE` 能力。

### 5.3 网络要求

- 内网机器需可达 MySQL 端口
- 若走 SSH 隧道，请先确认隧道稳定后再执行脚本


## 6. 可配置环境变量

以下变量都可在内网环境覆盖默认值。

| 变量名 | 默认值 | 说明 |
| --- | --- | --- |
| `MYSQL_DEMO_CONF` | `127.0.0.1,3308,demo_db,dev,123456,utf8mb4` | MySQL 连接串 |
| `MYSQL_HOST` | `127.0.0.1` | MySQL 主机 |
| `MYSQL_PORT` | `3308` | MySQL 端口 |
| `MYSQL_DATABASE` | `demo_db` | MySQL 数据库 |
| `MYSQL_USER` | `dev` | MySQL 用户 |
| `MYSQL_PASSWORD` | `123456` | MySQL 密码 |
| `MYSQL_CHARSET` | `utf8mb4` | MySQL 字符集 |
| `REGRESSION_TARGET_SCHEMA` | `rrs_test_dev` | 回归目标 schema |
| `REGRESSION_OLD_SCENARIO_ID` | `custTransInfo#old#20260309_01` | 旧版场景 |
| `REGRESSION_NEW_SCENARIO_ID` | `custTransInfo#new#20260309_01` | 新版场景 |
| `REGRESSION_BATCH_CODE` | `REGRESSION_DEMO_20260309_01` | 回归任务编码 |
| `REGRESSION_BATCH_NAME` | `custTransInfo 回归演示任务` | 回归任务名称 |


## 7. 运行步骤

说明：以下入口脚本都支持直接执行（`python3 scripts/*.py`），不依赖 `PYTHONPATH`。

### 7.1 设置环境变量

```bash
export MYSQL_DEMO_CONF='127.0.0.1,3308,demo_db,dev,123456,utf8mb4'
export REGRESSION_TARGET_SCHEMA='rrs_test_dev'
export REGRESSION_OLD_SCENARIO_ID='custTransInfo#old#20260309_01'
export REGRESSION_NEW_SCENARIO_ID='custTransInfo#new#20260309_01'
export REGRESSION_BATCH_CODE='REGRESSION_DEMO_20260309_01'
export REGRESSION_BATCH_NAME='custTransInfo 回归演示任务'
```

### 7.2 生产初始化（仅建表，不造数）

```bash
cd /path/to/json-diff
source .venv/bin/activate
python3 scripts/init_schema_only.py
```

### 7.3 执行回归比对（不清表）

默认建议：使用稳定 3 参数 CLI。

```bash
python3 scripts/regression_cli.py \
  "ALL" \
  "custTransInfo#old#20260309_01" \
  "custTransInfo#new#20260309_01"
```

说明：

- `regression_cli.py` 的 3 个参数固定为：`api_paths`、`old_scenario_id`、`new_scenario_id`。
- `api_paths=ALL` 或 `*` 表示按 `scenario_id` 覆盖的全部接口批量对比。
- `api_paths` 也可传逗号分隔列表做多接口子集对比。
- 脚本会自动生成 `batch_code`，报告默认输出到 `output/<batch_code>/测试报告.md`，并写 `output/latest.md`。

预检查（推荐先执行）：

```bash
python3 scripts/regression_cli.py \
  "ALL" \
  "custTransInfo#old#20260309_01" \
  "custTransInfo#new#20260309_01" \
  --dry-run
```

JSON 输出（便于平台解析）：

```bash
python3 scripts/regression_cli.py \
  "ALL" \
  "custTransInfo#old#20260309_01" \
  "custTransInfo#new#20260309_01" \
  --json
```

多接口子集示例：

```bash
python3 scripts/regression_cli.py \
  "/aml/wst/custTransInfo,/aml/wst/riskSummary" \
  "custTransInfo#old#20260309_01" \
  "custTransInfo#new#20260309_01"
```

## 9. 常见问题

### 9.1 连接失败

现象：`Can't connect to MySQL server`

排查：

- 检查 `MYSQL_DEMO_CONF` 主机和端口
- 检查隧道是否已建立
- 检查防火墙策略

### 9.2 认证失败

现象：`Access denied`

排查：

- 检查账号密码
- 检查用户授权范围（host + schema）

### 9.3 依赖安装失败

现象：内网无法下载 pip 包

排查：

- 确认已按第 4 节准备 wheelhouse
- 使用 `--no-index --find-links` 离线安装

### 9.4 报告为空或统计异常

排查：

- 确认脚本是否执行到了 `run_regression`（`regression_cli.py` / `run_regression_only.py`）
- 先执行 `--dry-run` 看预检查结果
- 检查 `scenario_id` 是否与造数一致
- 检查 `REGRESSION_TARGET_SCHEMA` 是否正确


## 10. 上线建议

如果要在内网长期运行（非演示），建议：

1. 固定使用 `init_schema_only.py` 与 `regression_cli.py` 分离执行。
2. 将配置改为 `.env` 或配置文件统一管理。
3. 将任务执行日志、报错和关键统计接入监控。
4. 将账号权限收敛为最小权限。
