# 内网部署与环境配置说明

## 1. 文档定位

本文档保留内网部署约束、权限和配置说明。  
所有运行命令（初始化、回归、测试、演示）统一维护在仓库根目录 [README.md](../README.md)。

## 2. 当前实现边界（按最新代码）

- 对比引擎：`DeepDiff(ignore_order=True)`
- 稳定入口：`scripts/regression_cli.py`
- Trace 模式：`--old-trace-id` + `--new-trace-id`
- 模糊匹配：`--fuzzy`（按 path 前缀）
- 兼容入口：`scripts/run_regression_only.py`（计划于 `2026-06-30` 弃用）
- 生产初始化入口：`scripts/init_schema_only.py`
- DEMO 入口：`scripts/init_demo_data.py`、`scripts/run_demo.py`

失败口径（报告与 trace_id 分类）：

- `compare_status != SUCCESS` 或 `diff_level = BLOCK` 视为失败

## 3. 环境要求

- 操作系统：Linux 或 macOS
- Python：`3.10+`（已在 `3.12` 验证）
- MySQL：`8.x`（`5.7` 可尝试，但未完整回归）

`requirements.txt` 主要依赖：

- `deepdiff==8.6.1`
- `PyMySQL>=1.1.1`
- `mcp`（仅 `scripts/regression_mcp_server.py` 场景需要）

## 4. MySQL 配置说明

支持两种方式：

1. 单串：`MYSQL_DEMO_CONF=host,port,database,user,password[,charset]`
2. 拆分：`MYSQL_HOST/MYSQL_PORT/MYSQL_DATABASE/MYSQL_USER/MYSQL_PASSWORD/MYSQL_CHARSET`

目标 schema：

- `REGRESSION_TARGET_SCHEMA`（默认：`rrs_test_dev`）

如果你使用本仓库配置文件 `conf/aml_conf.conf`，可在 `README` 查看一键导出示例。

## 5. 数据库权限要求

运行账号至少需要：

- 在基础库（如 `demo_db`）：`CREATE`（用于创建目标 schema）
- 在目标 schema（如 `rrs_test_dev`）：
  - `SELECT`
  - `INSERT`
  - `UPDATE`
  - `DELETE`
  - `CREATE`
  - `DROP`
  - `INDEX`
  - `ALTER`

若目标 schema 由 DBA 预创建，可按最小权限收敛。

## 6. 清表风险提示

仅以下脚本会触发清表：

- `scripts/init_demo_data.py`（默认清表，`--no-truncate` 可关闭）
- `scripts/run_demo.py`（演示一键流程，会清表）

不要在生产业务库直接执行清表型脚本。

## 7. 验证产物

每次执行完成后，报告输出为：

- `output/<batch_code>/测试报告.md`
- `output/latest.md`

报告已按最新实现包含：

- 成功/失败场景统计
- 成功/失败 trace_id 列表
- 失败请求明细（含 `query_params` 与 `request_body`）

## 8. 常见误用

- 把 `scenario_id` 传给 `--old-trace-id/--new-trace-id`（错误）
- 在场景里用不存在的 `api_path` 过滤导致 `selected=0`
- 忘记设置 `REGRESSION_TARGET_SCHEMA` 导致读写到非预期 schema
